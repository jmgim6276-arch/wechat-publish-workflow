# Style Iteration

This file captures the style lessons from the actual iteration on Tang's article.

## Core Rule

Cover and正文 are different systems.

- cover solves recognition
-正文 solves reading completion

Do not debug them as one thing.

## Iteration Order

When the user says the style is "not right", change in this order:

1. background tone
2. typography
3. section rhythm
4. micro design elements
5. content density

Do not change all five at once.

## What Worked Better Here

### Background

These worked better than dark black:

- `#F2F1EC`
- `#F5F3EE`

Paper-like off-white with light texture felt more refined than flat white or pure black.

### Typography

Better direction:

- serif-led title and section hierarchy
- readable正文 with a coherent Chinese font stack

Avoid making only the title beautiful while leaving the rest in a generic system font feel.

### Section Design

Good rule:

- one design grammar only

Example:

- use a single thin vertical line for `h2`
- do not stack dot + line + block at the same time

Too many accents quickly feel like a template.

### Pull Quotes

Better than highlighted boxes:

- large whitespace
- line break
- stronger sentence rhythm

Example pattern:

```text
我不是不喜欢技术，

我只是被一种错误的工作方式耗尽了。
```

### Lists

If the goal is refined minimalism:

- remove default bullet dots
- let spacing and indentation do the work

### Micro Elements

Use very few.

Good options:

- a short centered section divider
- one very small tag-like widget
- one quiet highlight treatment

Avoid mixing all of these aggressively.

## Common Failure Modes

### Too formal

Symptoms:

- too many full explanations
- article sounds like a structured essay, not a personal公众号 post

Fix:

- cut repeated explanations
- keep judgment sentences
- keep scene sentences

### Too decorative

Symptoms:

- h2 looks like a component block
- too many visual accents
- template feeling

Fix:

- remove one whole layer of decoration
- enlarge spacing instead of adding more shapes

### Too dense

Symptoms:

- people won't finish reading
- every paragraph contains a valid point, but the whole article feels heavy

Fix:

- reduce 15% to 20%
- cut explanation before cutting insight

## Practical Review Questions

When reviewing a style iteration, ask:

1. Is the problem background, typography, rhythm, or decorative grammar?
2. Does this page feel like a personal writer or a polished template?
3. Are the visual pauses enough?
4. Is the page still over-explaining with text?
